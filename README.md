## 🎯 Por Que Este Compromisso Existe

> *"A meia-vida de uma habilidade técnica hoje é de apenas 5 anos. Se o time não evolui continuamente, seu produto está morto antes do próximo release."*

Na **RET Tecnologia**, entendemos que o capital humano é o único ativo que se valoriza com o uso. Não tratamos desenvolvimento profissional como benefício corporativo genérico — tratamos como **investimento estratégico com ROI mensurável**.

### 📈 Os dados são inequívocos:

| Indicador (Pesquisa Global 2026) | Impacto | Fonte |
|---|---|---|
| Receita por colaborador em empresas com L&D forte | **+218%** | ATD Research |
| Ganho de produtividade via upskilling contínuo | **+39%** | WEF Future of Jobs |
| Melhoria na retenção de talentos | **+57%** | LinkedIn Workforce Learning |
| Redução de custos vs. contratação externa | **até 53%** | Deloitte Human Capital |
| Custo de substituição de engenheiro sênior | **150-200% do salário** | SHRM Benchmarks |
| Trabalhadores que precisam de reskilling até 2027 | **60%** | World Economic Forum |

> Cada engenheiro que sobe de nível eleva a qualidade de cada linha de código entregue aos nossos clientes. É um investimento com retorno **comprovado**.

### 2. 🧑‍🏫 Mentoria Bidirecional Estruturada

Profissionais seniores guiam o desenvolvimento de colegas em sessões 1:1 estruturadas. A mentoria é **bidirecional** — quem ensina, aprende. Quem aprende, acelera.

Seguimos o modelo de **mentorship programs** citado por Google (Googler-to-Googler) e Siemens como best practice para retenção:

- 📅 Sessões quinzenais de 45 minutos com **agenda documentada**
- 🎯 Pareamento mentor-mentorado por stack e objetivos de carreira
- 📊 Feedback contínuo e plano de evolução com **milestones trimestrais**
- 🔄 Reverse mentoring: juniors ensinam seniors sobre novas tecnologias

### 3. 💡 Acesso a Recursos de Ponta + Budget Dedicado

Cada membro do time tem acesso ilimitado a plataformas de ensino de classe mundial e um **budget pessoal** para investir em sua evolução:

| Plataforma | Uso | Diferencial |
|-----------|-----|-------------|
| **Alura** | Trilhas em português, infra a cloud | Ecossistema BR |
| **Pluralsight** | Role-based paths + Skill IQ assessments | Skill tracking nativo |
| **Coursera** | Especializações Stanford, Google, AWS | Universidades tier-1 |
| **O'Reilly** | Livros técnicos, early releases, live events | Catálogo referência |
| **Cloud Academy** | Labs interativos AWS/GCP/Azure | Hands-on real |
| **Budget pessoal** | Livros, conferências, certificações à escolha | **R$ 3.000+/ano** |

### 4. 💬 Guildas de Tecnologia (Comunidades de Prática)

Comunidades internas organizadas por especialidade — espaços seguros para inovação coletiva, modeladas no formato de **Communities of Practice** usado por Spotify e ThoughtWorks.

| Guilda | Foco | Key Topics 2026 |
|--------|------|-----------------|
| ⚙️ **Backend** | Arquitetura, APIs, microsserviços | Event-driven, gRPC, DDD |
| 🎨 **Frontend** | Performance, UX, design systems | Server Components, Edge |
| 🔧 **DevOps** | CI/CD, IaC, observabilidade | GitOps, DORA metrics |
| 🔒 **Security** | SAST/DAST, threat modeling, red team | AI Security, Supply Chain |
| 🤖 **AI & ML** | LLMs, RAG, AI Engineering | AI Quotient organizacional |

> **Atividades:** Tech talks semanais · Pair programming · Code reviews cruzados · Hackathons internos · Lightning talks de 10 min

### 5. 📐 Medição Baseada em DORA Metrics + L&D KPIs

Não medimos apenas *horas de treinamento* (vanity metric). Medimos **impacto no negócio** alinhado com DORA Metrics e Capability Velocity:

| Categoria | KPI | O que mede |
|-----------|-----|-----------|
| 🚀 **Velocidade** | Deployment Frequency | Frequência de deploys após upskilling |
| ✅ **Qualidade** | Change Failure Rate | Redução de falhas após treinamento |
| 🧠 **Competência** | Time to Competency | Tempo para atingir proficiency em nova stack |
| 🎯 **Retenção** | Skills Gap Closure Rate | Velocidade de fechamento de gaps |
| 📊 **Aplicação** | Return on Expectation (ROE) | Alinhamento entre resultado e expectativa |

## 🤝 Como Contribuir

Contribuições são bem-vindas! Se você quer sugerir melhorias, adicionar recursos ou trilhas de aprendizagem:

1. **Fork** este repositório
2. Crie uma **branch** com sua feature (`git checkout -b feature/nova-trilha`)
3. **Commit** suas mudanças (`git commit -m 'feat: adiciona trilha de Platform Engineering'`)
4. **Push** para a branch (`git push origin feature/nova-trilha`)
5. Abra um **Pull Request**

> 💡 **Dica:** Siga [Conventional Commits](https://www.conventionalcommits.org/) para mensagens de commit.
